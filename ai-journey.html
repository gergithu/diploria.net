Project Context & Notes

This section details the architectural choices and context behind the interactive application, reflecting the structural analysis requested in the initial brief.

Information Architecture Rationale

  • **Structure:** The design uses a **Narrative Dashboard** approach (linear scroll). This respects the original bio's chronological storytelling while making individual data points (Project Stages, AI Tool Comparisons) dynamically accessible.
  • **Content Safety:** High-risk details (e.g., "police bodycam footage") were generalized in the data array to "**digital media files (video and audio)**" to maintain confidentiality, as previously discussed.
  • **User Flow:** The flow is designed to answer the three core questions an employer/client would ask: **What did you do?** (Project Hub), **How did you do it?** (AI Toolkit), and **What was the result?** (Results & Learnings).
  • **Transferable Domains (MOVED UP):** The Domains section was moved directly after the Project Hub to immediately establish the broad relevance of the skills mastered.

Visual & Interactive Components

  • **Project Hub:** Uses a simple HTML/CSS list on the left to show the **process steps**, avoiding complex dependencies like Mermaid. Vanilla JS handles the state change on the right details panel. **Refinement:** Updated stage titles (e.g., "Simulated Negotiation") and highlighted the use of Adobe AI for data extraction.
  • **AI Toolkit (NEW):** Replaced the numerical chart with a **qualitative card** system to present the user's subjective truths about each model's traits without arbitrary numbers.
  • **Core Learnings (NEW):** A fourth learning on **Context Maintenance** has been added to reflect the necessity of managing LLM memory decay and authoritative hallucination.
  • **Results:** Added a card for **Resilience** to quantify the personal outcome of the project.
